# Soporte para mi-website-personal

Antes de abrir un issue revisa estas vías:

## 1. Documentación

- `README.md` para una visión general del proyecto.
- `docs/` para guías y referencias técnicas (si existen).
- `docs/adr/` para entender las decisiones de arquitectura.

## 2. Discusiones

Para preguntas y conversaciones de uso utiliza la sección
[Discussions del repositorio](https://github.com/Alexendros/mi-website-personal/discussions)
si está habilitada.

## 3. Issues

Abre un issue cuando tengas un problema reproducible o una propuesta
concreta. Sigue las plantillas del repositorio
(`.github/ISSUE_TEMPLATE/`).

## 4. Vulnerabilidades

No abras issues públicos. Sigue [SECURITY.md](SECURITY.md).

## 5. Contacto comercial

Para asuntos comerciales o de licencia escribe a contacto@alexendros.me.

## Tiempos de respuesta esperados

- Discussions y preguntas: cuando podamos, sin SLA.
- Issues con etiqueta `bug`: revisión inicial en 7 días naturales.
- Pull requests: primera respuesta en 7 días naturales.
- Vulnerabilidades: ver `SECURITY.md`.
